## 一、改进背景

原有项目启动流程繁琐，需手动打开多个终端窗口分别执行 `mvn spring-boot:run` 和 `npm run serve`，且 Maven 环境变量未配置导致启动失败。同时项目缺乏系统架构文档和技术演进规划。

## 二、本次改进内容

### 2.1 🚀 新增启动脚本（替代手动操作）

#### `start.bat`（双击启动）
- **定位**：Windows 用户双击即可运行
- **功能**：提供菜单式交互，支持 4 种启动模式
- **关键修复**：脚本头部自动注入 Maven 路径，无需依赖系统环境变量

```
[1] 同时启动后端 + 前端
[2] 仅启动后端
[3] 仅启动前端
[4] 安装依赖并同时启动后端 + 前端
```

#### `start.ps1`（命令行启动）
- **定位**：PowerShell 用户，支持参数化调用
- **使用示例**：

```powershell
.\start.ps1               # 同时启动
.\start.ps1 -Backend      # 仅后端
.\start.ps1 -Frontend     # 仅前端
.\start.ps1 -All -Install # 安装依赖 + 启动
```

#### 🔧 解决的关键问题：Maven 找不到

| 问题 | 方案 |
|------|------|
| `mvn` 命令不可用 | 系统已安装但未重启终端 |
| 环境变量刚添加未生效 | 两个脚本均显式注入 `%MAVEN_HOME%\bin` 到 PATH |
| 新 cmd 窗口继承问题 | `start.ps1` 在父进程 PATH 注入后传递给子进程 |
