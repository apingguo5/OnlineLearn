# 登录跳转问题修复报告

## 问题描述

登录成功后无法跳转到对应的用户界面，页面停留在登录页无反应。

## 问题分析

### 原因一：角色判断逻辑脆弱（已修复）

`frontend/src/views/login/Login.vue` 中登录成功后的路由跳转代码：

```javascript
// 原代码：三个独立 if，无兜底处理
if (data.resultData.roleId == 1) {
    this.$router.push("/adminHome")
}
if (data.resultData.roleId == 2) {
    this.$router.push("/teacherHome")
}
if (data.resultData.roleId == 3) {
    this.$router.push("/studentweb")
}
```

**缺陷**：使用三个独立 `if` 而非 `else if` 链。当 `roleId` 为 `null` / `undefined` 或非 1/2/3 的值时，没有任何 `if` 条件成立，导致 `this.$router.push(...)` 永远不会执行，页面停留在登录页。

### 原因二：新注册用户 roleId 为空（待完善）

测试新注册的账号登录时，后端返回的 `roleId` 为 `undefined`，说明注册流程未正确关联用户角色。

## 已完成的修复

### 1. 路由跳转逻辑优化（`Login.vue`）

- 将三个独立 `if` 改为 `else if` 链，确保只有一条路径执行
- 添加 `else` 兜底分支，当 roleId 异常时给出提示（显示具体 roleId 值）
- 提取 `result` 中间变量使代码更清晰可读
- "登录成功"提示移至跳转前，确保用户立即收到反馈

```javascript
const result = data.resultData;
const roleId = result.roleId;

if (roleId == 1) {
    this.$router.push("/adminHome")
} else if (roleId == 2) {
    this.$router.push("/teacherHome")
} else if (roleId == 3) {
    this.$router.push("/studentweb")
} else {
    // 兜底提示，方便排查
    this.$message({
        message: '未配置角色权限（roleId=' + roleId + '），请联系管理员',
        type: 'warning'
    });
}
```

### 2. 待完善：注册流程增加角色选择

当前注册接口（`/study/user/registry`）将新用户固定为 `roleId=3`（学生），但用户无法在注册时选择身份角色，且未关联班级/单位。

**建议优化方向**：
- 注册表单增加角色选择（学生/教师）和班级选择
- 根据所选角色动态注册对应的 `user_role` 关联
- 管理员可后续审核并修改用户角色
