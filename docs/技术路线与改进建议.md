## 3. 需优先改进的问题

按优先级从高到低排列：

### 🔴 P0 — 关键安全与稳定性问题

#### 3.1 ⚠️ 无安全框架（最严重）

**问题描述**：当前项目**没有使用 Spring Security / Shiro** 或其他安全框架，权限控制完全由前端 Cookie 中的 `userId` 实现（`router.beforeEach` 检查），后端接口没有任何拦截校验。

- **风险**：任何用户只要知道后端 `api/xxx/xxx` 接口地址，即可直接调用；伪造 Cookie 中的 `userId` 可越权访问其他用户数据
- **影响范围**：所有后端 Controller 接口
- **建议方案**：
  1. **短期**：后端添加拦截器，对所有非登录接口校验 Token（JWT 或 Session）
  2. **中期**：引入 Spring Security + JWT，实现基于 RBAC（Role-Based Access Control）的精细化权限管理

#### 3.2 后端代码与 Java 25 不兼容风险

**问题描述**：`pom.xml` 中 `java.version` 设为 `1.8`，但实际运行在 Java 25。虽然向后兼容，但 Spring Boot 2.5.6 官方支持的 JDK 上限为 JDK 17，后续依赖可能因模块系统（JPMS）限制而出现问题（如 `javax` → `jakarta` 命名空间变更）。

- **建议方案**：
  1. 升级 `java.version` 到 `17`（LTS 版本，匹配 Spring Boot 2.5 支持上限）
  2. 明确 JVM 版本要求，避免 JDK 11 / 17 / 21 / 25 混用导致的不确定性

#### 3.3 无日志框架

**问题描述**：MyBatis SQL 日志直接输出到控制台（`log-impl: org.apache.ibatis.logging.stdout.StdOutImpl`），没有使用 SLF4J / Logback 等专业日志框架。

- **影响**：无法按级别过滤日志、无法文件滚动、无法集中管理
- **建议方案**：
  1. 移除 `log-impl: stdout`，改用 SLF4J + Logback 配置
  2. 控制台日志级别设为 `WARN`，将 `DEBUG` 日志输出到独立文件
  3. 日志配置参考 `logback-spring.xml`

#### 3.4 配置文件中数据库密码明文存储

**问题描述**：`application.yml` 中数据库 `password` 直接明文写死。

- **建议方案**：
  1. 使用环境变量 `${MYSQL_PASSWORD}` 引用
  2. 使用 Spring Cloud Config 或 Jasypt 加密配置文件
  3. 将 `application.yml` 加入 `.gitignore`，提交模板文件

### 🟡 P1 — 代码质量与可维护性

#### 3.5 DTO 放在 Controller 包中

**问题描述**：`VideoWatchRecordDto.java` 直接放在 `controller` 包下，不符合分层规范。DTO（数据传输对象）应放在独立的 `dto` 包中。

- **建议**：在 `com.rabbiter.ol` 下新建 `dto` 包，将 `VideoWatchRecordDto` 移入该包

#### 3.6 路由配置冗余

**问题描述**：前端 `router/index.js` 中存在重复的路由定义（如 `applicanthistory` 定义了两次），且所有路由都使用了相同的 `meta.requireAuth: true` 守卫逻辑，可以批量简化。

- **建议**：清理重复路由，必要时可使用路由配置工厂函数生成

#### 3.7 全局异常处理不够精细

**问题描述**：`GlobalExceptionHandler` 存在但未覆盖所有异常类型，如参数校验异常（`MethodArgumentNotValidException`）、HTTP 请求方法不支持异常（`HttpRequestMethodNotSupportedException`）。

- **建议**：完善全局异常处理器，针对不同异常类型返回有意义的 HTTP 状态码和错误消息

#### 3.8 缺少 Service 层接口注释

**问题描述**：所有 Service 接口和实现类均无 JavaDoc 注释，影响代码理解和维护。

- **建议**：补全 Controller、Service、Entity 等关键类的 JavaDoc 注释

### 🟢 P2 — 功能增强与性能优化

#### 3.9 缺少 Redis 缓存

**问题描述**：项目完全依赖 MySQL，没有使用缓存。频繁访问的热点数据（如课程列表、用户信息）每次都查询数据库。

- **建议**：引入 Redis 缓存热门课程数据、用户 Session / Token 信息
  - 首页课程列表 → 缓存 5 分钟
  - 用户信息 → 缓存 30 分钟（变更时主动失效）

#### 3.10 缺少 WebSocket 实时推送

**问题描述**：作业批改、问答回复等都需要用户手动刷新页面才能看到最新结果。

- **建议**：引入 WebSocket（或 Server-Sent Events），实现：
  - 作业批改后实时通知学生
  - 问答有新回复时实时推送
  - 管理员/教师审批申请时实时推送

#### 3.11 文件上传功能不完善

**问题描述**：虽然配置了 1024GB 的 `max-file-size`，但上传的文件存储在本地文件系统，不具备多服务器共享、备份、CDN 分发能力。

- **建议**：
  - 小文件（头像、附件）→ 上传至 OSS（阿里云 / 腾讯云 / MinIO）
  - 大文件（视频）→ 分片上传 + 断点续传
  - 非对称目录存储，避免单目录文件过多

#### 3.12 视频播放功能局限

**问题描述**：使用 `video.js` 播放本地视频文件，未使用流媒体协议（HLS / DASH），不支持：
  - 自适应码率
  - 视频加密（DRM）
  - 拖动进度条时的快速加载

- **建议**：
  - 视频转码后使用 HLS 协议播放
  - 引入阿里云视频点播 / 七牛云 / M3U8 方案

#### 3.13 前端路由导航守卫安全性不足

**问题描述**：前端只通过 Cookie 中是否存在 `userId` 来判断是否登录，没有校验该 `userId` 的合法性，也没有结合角色做权限路由。

- **建议**：
  - 每次页面加载时向后端发起 Token 校验请求
  - 路由守卫中结合用户角色（admin / teacher / student）进行权限路由过滤
  - 使用 `vue-router` 动态路由注册实现角色专属菜单

#### 3.14 缺少单元测试

**问题描述**：`spring-boot-starter-test` 依赖存在（在 `pom.xml` 中），但项目中看不到任何单元测试。

- **建议**：
  - 后端：为 Service 层和 Controller 层编写 JUnit 5 + Mockito 单元测试
  - 前端：引入 Jest / Vitest 编写组件测试
  - 最低目标：关键业务逻辑（登录、权限校验、成绩计算）测试覆盖率 > 70%

#### 3.15 MySQL 连接未使用连接池配置

**问题描述**：`application.yml` 中只有基础数据源配置，没有配置 HikariCP 连接池参数（默认值可能不适合生产环境）。

- **建议**：添加 HikariCP 配置参数：

```yaml
spring:
  datasource:
    hikari:
      maximum-pool-size: 20
      minimum-idle: 5
      idle-timeout: 300000
      connection-timeout: 20000
      max-lifetime: 1200000
```

## 5. 总结

### 必须立即处理（P0）

1. **添加后端权限校验**（拦截器 / Spring Security） — 无此功能项目无法上线
2. **数据库密码加密或环境变量化** — 安全合规基本要求
3. **引入日志框架** — 替换 `stdout` 控制台日志

### 建议近期处理（P1）

4. 明确 Java / Spring Boot 版本并升级
5. 完善全局异常处理
6. 优化代码结构（DTO 包、路由去重）
7. 补全 JavaDoc 注释

### 规划中考虑（P2+）

8. Redis 缓存、WebSocket 推送、文件存储到 OSS
9. 前端 Vue 3 升级、TypeScript 迁移
