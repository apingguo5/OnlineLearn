# OnlineLearn 环境管理设计文档

## 1. 设计目标

解决项目在多开发人员、多机器环境下 **环境配置分散、硬编码、难以迁移** 的问题。

### 现有问题

| 问题 | 现象 | 影响 |
|------|------|------|
| 路径硬编码 | `start.bat` 中写死 `MAVEN_HOME` | 每换一台机器就要改脚本 |
| 配置分散 | JDK 路径、数据库连接等信息散落在多个文件中 | 没有统一入口，容易遗漏 |
| 无版本检测 | 启动前不检查环境 | 启动失败后排查困难 |
| 本地配置与模板混用 | 修改后误提交到仓库 | 影响其他开发人员 |

## 2. 整体架构 — 单一数据源设计

```
OnlineLearn/
├── env/                          # 环境管理目录
│   ├── .env                     # [⭐ 唯一数据源] 所有环境变量的默认模板（提交仓库）
│   ├── .env.local               # [本地覆盖] 复制自 .env，修改本地路径（已 .gitignore）
│   ├── env-config.bat           # [加载器] CMD：从 .env / .env.local 读取配置
│   ├── env-config.ps1           # [加载器] PowerShell：从 .env / .env.local 读取配置
│   ├── init-env.bat             # [CMD] 环境初始化 + 检测脚本
│   ├── init-env.ps1             # [PS] 环境初始化 + 检测脚本
│   ├── check-env.bat            # [CMD] 环境健康检查脚本
│   └── check-env.ps1            # [PS] 环境健康检查脚本
├── start.bat                    # [重构] CMD 启动脚本（引用 env-config.bat）
├── start.ps1                    # [重构] PS 启动脚本（引用 env-config.ps1）
└── .gitignore                   # 排除 .env.local
```

### 核心设计思想

> **所有环境变量只在一个地方定义 — `env/.env`（模板），个人覆盖写在 `env/.env.local`（本地）。**
>
> 修改环境配置时，**修改一处就同时影响 CMD 和 PowerShell**，不再需要维护两套配置。

## 3. 环境配置项说明

### 3.1 JDK 配置

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| `JAVA_HOME` | JDK 安装根目录 | `D:\Environment\jdk1.8.0_202` |

- 项目要求 JDK 1.8+

### 3.2 Maven 配置

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| `MAVEN_HOME` | Maven 安装根目录 | `D:\Environment\maven\apache-maven-3.9.14` |

- 项目要求 Maven 3.x
- pom.xml 中已配置阿里云镜像仓库，国内下载依赖更快

### 3.3 Node.js 配置

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| `NODE_HOME` | Node.js 安装目录 | `C:\Program Files\nodejs` |

- 项目要求 Node.js 12+
- Node 17+ 自动添加 `NODE_OPTIONS=--openssl-legacy-provider` 兼容参数

### 3.4 MySQL 配置

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| `MYSQL_HOST` | 数据库主机 | `192.168.1.38` |
| `MYSQL_PORT` | 数据库端口 | `3306` |
| `MYSQL_DATABASE` | 数据库名 | `online_learn` |
| `MYSQL_USER` | 用户名 | `newuser` |
| `MYSQL_PASSWORD` | 密码 | `yourpassword` |

**注意**：数据库配置需要同步修改 `backend/src/main/resources/application.yml`

### 3.5 端口配置

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `BACKEND_PORT` | `9251` | 后端服务端口 |
| `FRONTEND_PORT` | `9252` | 前端服务端口 |

### 3.6 构建选项

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `SKIP_TESTS` | `true` | Maven 构建是否跳过测试 |
| `NPM_INSTALL_ARGS` | `--legacy-peer-deps` | npm install 额外参数 |

## 4. `env/.env` 文件格式

```
# 这是一个标准的 KEY=VALUE 格式文件
# 注释行以 # 开头
# 空行会被忽略
# 支持 = 号两侧有空格（会被 trim）

JAVA_HOME=D:\Environment\jdk1.8.0_202
MAVEN_HOME=D:\Environment\maven\apache-maven-3.9.14
MYSQL_HOST=192.168.1.38
```

- `.env` 是 **模板文件**，提交到仓库，包含合理的默认值
- `.env.local` 是 **本地覆盖文件**，从 `.env` 复制而来，不提交到仓库
- 运行时优先加载 `.env.local`，不存在时降级为 `.env`

## 5. 使用流程

### 首次使用（新开发人员）

```
1. 克隆仓库
   git clone <repo-url>
   cd OnlineLearn

2. 初始化环境（自动从 .env 复制生成 .env.local）
   env\init-env.bat           (CMD)
   或
   .\env\init-env.ps1         (PowerShell)

3. 编辑本地配置
   📝 编辑 env\.env.local
   修改 JAVA_HOME / MAVEN_HOME / NODE_HOME / MYSQL_* 为您本机路径

4. 检查环境
   env\check-env.bat          (CMD)
   或
   .\env\check-env.ps1        (PowerShell)

5. 启动项目
   start.bat                   (菜单模式)
   或
   start.bat all               (一键启动)
```

### 日常使用

```bash
# 同时启动后端 + 前端
start.bat all
# 或
.\start.ps1 -All

# 仅启动后端
start.bat backend
# 或
.\start.ps1 -Backend

# 仅启动前端
start.bat frontend
# 或
.\start.ps1 -Frontend

# 安装依赖并启动
start.bat install
# 或
.\start.ps1 -All -Install
```

## 6. 环境配置加载机制

### 加载优先级

启动脚本按以下顺序加载配置：

1. `env/.env.local` — 本地个性化配置（**优先**）
2. `env/.env` — 默认模板配置（**降级**）
3. 系统 `PATH` 环境变量 — 均未找到时使用（降级）

### 加载流程

```
start.bat
  │
  └── call env\env-config.bat
        │
        ├── 存在 .env.local?  ──Yes──> 读取 .env.local（本地覆盖）
        │     No
        └── 存在 .env?        ──Yes──> 读取 .env（模板）
              No
              └── 输出警告（未找到环境配置文件）
```

### 关键设计

```
env/.env  (模板, 提交仓库, 包含默认值)
    │
    ├── 复制 → env/.env.local (本地, 已 gitignore)
    │
    └── 被读取 → 所有脚本统一从这里获取配置
```

**修改环境配置时**：
- 只编辑 `env/.env.local`
- CMD（`env-config.bat`）和 PowerShell（`env-config.ps1`）都会自动读取
- 无需两处分别修改

## 7. 为什么不推荐配置系统环境变量？

| 方式 | 优点 | 缺点 |
|------|------|------|
| **本项目方案**（env 目录配置） | 项目级隔离，不污染系统；每个项目可独立控制版本；新成员开箱即用 | 每次启动需执行脚本 |
| 配置系统环境变量 | 全局可用，无需额外操作 | 多项目 JDK/Maven 版本冲突；新成员需手动配；切换项目麻烦 |

**推荐场景**：
- 日常开发用本项目方案，每个项目独立控制版本
- 如果只做这一个项目，也可把配置写到系统环境变量，启动脚本会自动识别

## 8. 常见问题

### Q: 启动时提示 "java 不是内部或外部命令"

解决：检查 `env/.env.local` 中 `JAVA_HOME` 路径是否正确，指向 JDK 的安装目录（不是 JRE）。如果路径中有空格，确认没有多加或缺少引号。

### Q: Maven 构建速度慢

解决：pom.xml 中已配置阿里云镜像，如果还慢可以检查网络或手动下载依赖到本地仓库。

### Q: 前端启动报 OpenSSL 错误

解决：Node.js 17+ 版本会触发此问题，启动脚本会自动添加 `NODE_OPTIONS=--openssl-legacy-provider`。如果仍有问题，可以检查 `env/.env.local` 中 `NODE_HOME` 路径是否正确。

### Q: 修改环境配置后需要重启终端吗？

**不需要**。启动脚本每次运行都会重新读取 `env/.env.local` 或 `env/.env`，即时生效。

### Q: 如何切换 JDK 版本？

编辑 `env/.env.local` 修改 `JAVA_HOME` 路径即可，不影响系统全局配置。

### Q: .env 和 .env.local 有什么区别？

| 文件 | 用途 | 是否提交仓库 |
|------|------|-------------|
| `env/.env` | 默认模板，包含示例值 | ✅ 提交 |
| `env/.env.local` | 本机覆盖，修改为实际路径 | ❌ 不提交（已 .gitignore） |

### Q: 为什么不用 .env 直接覆盖，而要复制一份 .env.local？

因为 `.env` 会被 `git` 追踪。如果直接修改 `.env`，不小心就会把本地路径提交到仓库，影响其他开发人员。
