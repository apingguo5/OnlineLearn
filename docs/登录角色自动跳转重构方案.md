# 登录角色自动跳转重构方案

## 概述

本次重构改进了系统的登录流程，**去除了登录界面手动选择角色的操作**，改为后端根据用户 ID 自动查询 `user_role` 表获取真实角色，前端根据角色自动跳转到对应的功能页面。

## 重构背景

原系统在登录时需要用户手动选择角色（管理员/教师/学生），存在以下问题：
1. 用户体验差：需要多一步操作
2. 角色选择可能与实际权限不符（用户可能选错）
3. 前后端角色信息割裂，维护成本高

## 修改清单

### 1. 后端修改

#### 1.1 `UserDao.xml` — login SQL 简化

- **修改前**：login SQL 同时 `LEFT JOIN user_role` 联表查询角色
- **修改后**：login SQL 只做**用户验证**（查账号+密码+班级），删除角色相关 JOIN

```xml
<select id="login" resultType="java.util.HashMap">
    SELECT
    u.id AS userId,
    u.user_name AS userName,
    u.password AS password,
    uc.class_id AS classId
    FROM
    `user` u
    LEFT JOIN user_class uc ON u.id = uc.user_id
    <where>
        <if test="account != null">
            AND u.account = #{account}
        </if>
    </where>
    LIMIT 1
</select>
```

#### 1.2 `UserServiceImpl.java` — 角色独立查询

**修改前**：直接从 login 结果中读取 `roleId`

**修改后**：密码验证成功后，根据 `userId` 单独查询 `user_role` 表获取真实角色

```java
@Autowired
private UserRoleService userRoleService;  // 新增注入

if (passwordMatched) {
    user.remove("password");

    // 根据用户ID查询 user_role 表获取角色
    Number userIdNum = (Number) user.get("userId");
    Integer userId = userIdNum.intValue();
    List<UserRoleEntity> roleList = userRoleService.list(
        new QueryWrapper<UserRoleEntity>().eq("user_id", userId)
    );
    if (roleList != null && !roleList.isEmpty()) {
        user.put("roleId", roleList.get(0).getRoleId());
    }
    return users;
}
```

#### 1.3 修复的关键 BUG

- **类型转换错误**：MyBatis 对 `INT UNSIGNED` 字段返回的是 `Long` 类型（不是 `Integer`），直接 `(Integer) user.get("userId")` 会抛出 `ClassCastException` 导致 500 错误
- **修复方式**：使用 `Number` 作为安全中间类型转换 `(Number) user.get("userId")` → `.intValue()`
- **`LIMIT 1` 问题**：login SQL 缺少 `LIMIT 1` 时，若同一账号多角色（如既有 student 又有 teacher 角色），`user_role` 联表会返回多行，第一个不一定是正确角色。单独查询 `user_role` 表后此问题自然解决

### 2. 前端修改

#### 2.1 `Login.vue` — 删除角色选择

**修改前**：登录界面需要用户手动选择角色

**修改后**：登录界面仅为简洁的账号+密码输入框，底部提示"根据账号角色自动跳转对应页面"

![登录界面](登录不选择角色，账号密码登录即可)

**登录成功后的跳转逻辑**（无需修改，原代码已正确实现）：

```javascript
// Login.vue — login 方法
if (roleId == 1) {
    this.$router.push("/adminHome")          // 管理员 → 全局管理页面
} else if (roleId == 2) {
    this.$router.push("/teacherdashboard")   // 教师 → 班级管理工作台
} else if (roleId == 3) {
    this.$router.push("/studentweb")         // 学生 → 上课页面
}
```

#### 2.2 管理员专属侧边栏

**新增** `frontend/src/views/admin/aside/AdminAside.vue`

为管理员角色创建了专用的精简侧边栏，仅包含以下菜单：
- 管理员管理
- 学生管理
- 教师管理
- 课程管理

#### 2.3 教师 `Aside.vue` 菜单优化

将菜单调整为教师角色专属功能：
- 班级管理
- 知识点
- 作业
- 练习
- 视频

### 3. 注册功能

注册页面（抽拉式抽屉）**仍然保留角色选择**，因为注册时需要根据角色做不同的业务逻辑（如学生需要选择班级、教师不需要），这是合理的。

## 数据流图

```
用户输入账号密码
       ↓
后端 /study/user/login 接口
       ↓
UserDao.login() — 仅验证账号密码，查出 userId
       ↓
密码验证成功
       ↓
UserRoleService.list() — 根据 userId 查 user_role 表
       ↓
后端返回 { userId, userName, classId, roleId }
       ↓
前端 Cookies.set('roleId', roleId)
       ↓
根据 roleId 自动跳转
  ├── 1 → /adminHome (管理员)
  ├── 2 → /teacherdashboard (教师)
  └── 3 → /studentweb (学生)
```

## 注意事项

1. **部署时需要重启后端**，使 MyBatis Mapper 和 Java Service 修改生效
2. 前端热更新即可，无需额外操作
3. 数据库 `user_role` 表必须保证用户至少有一条角色记录，否则登录会返回无角色信息
