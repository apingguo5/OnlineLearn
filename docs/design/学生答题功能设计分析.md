
## 概述

本文档对整个答题系统进行多层次的分析与验证，涵盖数据库设计、后端API架构、前端页面实现、路由配置、状态管理等。系统支持单选题、多选题、判断题、填空题、文字题五种题型，功能包括教师上传习题、组卷、学生答题、暂存、提交、教师批改等完整闭环。

## 一、数据库表结构设计

### 1.1 question（题目表）

```sql
CREATE TABLE `question` (
  `id` int(11) NOT NULL AUTO_INCREMENT,
  `question_type` tinyint(4) DEFAULT NULL COMMENT '题型：1单选 2多选 3判断 4填空 5文字',
  `stem` text COMMENT '题干',
  `options` text COMMENT '选项（JSON格式）',
  `answer` varchar(500) DEFAULT NULL COMMENT '答案',
  `analysis` text COMMENT '解析',
  `score` int(11) DEFAULT 5 COMMENT '默认分值',
  `difficulty` tinyint(4) DEFAULT 2 COMMENT '难度：1-5',
  `course_id` int(11) DEFAULT NULL COMMENT '所属课程',
  `chapter_id` int(11) DEFAULT NULL COMMENT '所属章节',
  `creator_id` int(11) DEFAULT NULL COMMENT '创建者用户ID',
  `status` tinyint(4) DEFAULT 1 COMMENT '状态：0禁用 1启用',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP,
  `update_time` datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
)
```

**分析：** ✅ 设计合理。options使用JSON格式存储，兼容单选/多选的选项列表；question_type使用tinyint数字，便于扩展

### 1.2 exam_paper（试卷表）

```sql
CREATE TABLE `exam_paper` (
  `id` int(11) NOT NULL AUTO_INCREMENT,
  `title` varchar(200) NOT NULL COMMENT '试卷名称',
  `description` text COMMENT '考试说明',
  `paper_type` tinyint(4) DEFAULT 1 COMMENT '试卷类型：1普通考试 2随堂测验 3作业',
  `course_id` int(11) DEFAULT NULL COMMENT '所属课程',
  `chapter_id` int(11) DEFAULT NULL COMMENT '关联章节',
  `class_id` int(11) DEFAULT NULL COMMENT '指定班级（null为全部可见）',
  `creator_id` int(11) DEFAULT NULL COMMENT '创建者',
  `duration` int(11) DEFAULT 60 COMMENT '考试时长（分钟）',
  `total_score` int(11) DEFAULT 100 COMMENT '总分',
  `status` varchar(20) DEFAULT 'draft' COMMENT 'draft-草稿 published-已发布',
  `publish_time` datetime DEFAULT NULL COMMENT '发布时间',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP,
  `update_time` datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
)
```

**分析：** ✅ 设计合理。status使用varchar存储状态更直观；class_id支持按班级发布，flexible

### 1.3 exam_paper_question（试卷题目关联表）

```sql
CREATE TABLE `exam_paper_question` (
  `id` int(11) NOT NULL AUTO_INCREMENT,
  `paper_id` int(11) NOT NULL,
  `question_id` int(11) NOT NULL,
  `score` int(11) DEFAULT NULL COMMENT '该题在试卷中的分值（覆盖题目默认分值）',
  `sort_order` int(11) DEFAULT 0 COMMENT '排序',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP,
)
```

**分析：** ✅ 设计合理。支持每道题单独分值覆盖，sort_order控制题目顺序

### 1.4 student_answer_record（学生答题记录表）

```sql
CREATE TABLE `student_answer_record` (
  `id` int(11) NOT NULL AUTO_INCREMENT,
  `paper_id` int(11) NOT NULL,
  `question_id` int(11) NOT NULL,
  `student_id` int(11) NOT NULL,
  `answer` text COMMENT '学生答案',
  `score` int(11) DEFAULT 0 COMMENT '得分',
  `is_correct` tinyint(4) DEFAULT 0 COMMENT '是否正确',
  `review_status` tinyint(4) DEFAULT 0 COMMENT '批改状态：0未批改 1已批改',
  `remark` text COMMENT '教师评语',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP,
  `update_time` datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
)
```

**分析：** ✅ 设计合理。每条记录对应一题，支持逐题批改和暂存。review_status区分客观题（自动批改）和主观题（人工批改）状态

### 1.5 student_exam_record（学生考试记录表）

```sql
CREATE TABLE `student_exam_record` (
  `id` int(11) NOT NULL AUTO_INCREMENT,
  `paper_id` int(11) NOT NULL,
  `student_id` int(11) NOT NULL,
  `status` varchar(20) DEFAULT 'in_progress' COMMENT 'in_progress-答题中 submitted-已提交 graded-已批改',
  `total_score` int(11) DEFAULT 0 COMMENT '总分',
  `start_time` datetime DEFAULT NULL,
  `submit_time` datetime DEFAULT NULL,
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP,
  `update_time` datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
)
```

**分析：** ✅ 设计合理。独立记录学生考试整体状态，与答题明细分离，便于统计和查询
